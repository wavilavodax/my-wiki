# 修复检查（Repair Check）

> **标签:** #概念 #维护

## 什么是修复检查？

Karpathy 的 LLM Wiki 方法里，专门设计的一个维护机制。让 AI 定期扫描整个知识库，找出潜在问题并修复。

## 每周检查四类问题

1. 矛盾说法 — 哪些页面之间信息互相矛盾
2. 孤儿页面 — 哪些页面没有被任何其他页面链接
3. 悬空概念 — 哪些概念被反复提到但没有自己的页面
4. 信息过时 — 哪些内容被新素材推翻

## 怎么触发？

在 Claude Code 里说：
> "帮我检查一下 Wiki 的健康状况"

## 自动化

把这句话设成每周定时任务，自动跑。Wiki 每周自己做一次体检，自己修好，你什么都不用管。
